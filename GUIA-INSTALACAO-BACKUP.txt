PASSO 1: Instalar Dependência
─────────────────────────────────────────────────────────────

No terminal, na pasta clinica-web:

npm install node-cron --save


PASSO 2: Adicionar Código no server.js
─────────────────────────────────────────────────────────────

1. Abra o arquivo: server.js

2. Vá ATÉ O FINAL do arquivo

3. Procure a linha:
   app.listen(PORT, () => {

4. ANTES dessa linha, cole TODO o conteúdo do arquivo:
   codigo-backup-automatico.js

5. Salve o arquivo


PASSO 3: Adicionar Página de Gerenciamento
─────────────────────────────────────────────────────────────

1. Copie o arquivo: gerenciar-backups.html

2. Cole na pasta: clinica-web/public/

3. Estrutura final:
   clinica-web/
   ├── public/
   │   ├── index.html
   │   ├── login.html
   │   ├── gerenciar-backups.html  ← AQUI
   │   ├── script.js
   │   └── styles.css

PASSO 4: Atualizar package.json

PASSO 5: Criar .gitignore (IMPORTANTE!)
─────────────────────────────────────────────────────────────

Crie arquivo .gitignore na raiz:

*.db

# Node

# Uploads

PASSO 6: Remover .db do Git
─────────────────────────────────────────────────────────────

git rm --cached clinica.db
git rm --cached clinica-backup*.db
git add .gitignore
git commit -m "Adicionar sistema de backup e ignorar .db"


PASSO 7: Fazer Deploy

📊 ROTAS DA API
═══════════════════════════════════════════════════════════════

GET  /api/backups/list
→ Lista todos os backups disponíveis
→ Retorna: JSON com lista de backups

GET  /api/backups/download/:data
→ Baixa backup específico
→ Exemplo: /api/backups/download/2026-03-16

POST /api/backups/create
→ Cria backup manual
→ Retorna: JSON com sucesso/erro

POST /api/backups/restore/:data
→ Restaura backup (CUIDADO!)
→ Faz backup do atual antes de restaurar
→ Exemplo: /api/backups/restore/2026-03-16

═══════════════════════════════════════════════════════════════
🆘 SOLUÇÃO DE PROBLEMAS
═══════════════════════════════════════════════════════════════

PROBLEMA: Backup não está rodando
─────────────────────────────────────────────────────────────

SOLUÇÃO:
1. Verifique logs do Railway:
   railway logs

2. Procure por:
   "✅ Sistema de backup automático ativado!"
   "📋 Backup criado: clinica-backup-XXXX.db"

3. Se não aparecer, node-cron não foi instalado:
   npm install node-cron


PROBLEMA: Erro ao acessar /gerenciar-backups.html
─────────────────────────────────────────────────────────────

SOLUÇÃO:
1. Verifique se arquivo está em public/
2. Faça git push
3. Aguarde deploy
4. Limpe cache do navegador (Ctrl+Shift+R)


PROBLEMA: Lista de backups vazia
─────────────────────────────────────────────────────────────

SOLUÇÃO:
1. Aguarde até às 03:00 (backup automático)
2. OU clique em "💾 Criar Backup Agora"
3. Aguarde 1-2 segundos
4. Clique em "🔄 Atualizar Lista"


PROBLEMA: Restaurar não funciona
─────────────────────────────────────────────────────────────

SOLUÇÃO:
1. Verifique se está logado
2. Verifique conexão com internet
3. Tente baixar o backup primeiro
4. Depois restaure manualmente
